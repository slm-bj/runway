# Contributing to Ideas Collection Project

感谢你对本项目的兴趣！我们欢迎各种形式的贡献。

Thank you for your interest in this project! We welcome all forms of contributions.

## 贡献方式 (Ways to Contribute)

### 1. 提交创意 (Submit Ideas)

如果你有好的想法，请按照以下步骤提交：

1. **Fork 本仓库**
2. **创建新分支**: `git checkout -b idea/your-idea-name`
3. **在 `ideas/` 目录创建你的创意文件**: 参考 `ideas/TEMPLATE.md`
4. **提交更改**: `git commit -m "[IDEA] Add your idea title"`
5. **推送分支**: `git push origin idea/your-idea-name`
6. **创建 Pull Request**: 使用标题格式 `[IDEA] Your Idea Title`

**PR 描述应包含**:
- 创意的简要概述
- 为什么这个创意有价值
- 任何相关的背景信息

### 2. 投票支持创意 (Vote for Ideas)

- 浏览开放的 Pull Requests
- 对你喜欢的创意使用 👍 反应
- 在评论中提供建设性反馈

### 3. 实施项目 (Implement Projects)

如果你想实施一个获得高票的创意：

1. **在相关 Issue 或 PR 中表明意愿**
2. **与创意提出者和维护者讨论计划**
3. **创建项目仓库并开始开发**
4. **在 `projects/` 目录添加项目文档**:
   - `README.md` - 需求描述
   - `proposal.md` - 立项报告
   - `REPOSITORY.md` - 仓库链接
5. **提交 PR 更新项目列表**

### 4. 改进文档 (Improve Documentation)

- 修正拼写或语法错误
- 改进说明的清晰度
- 添加翻译
- 更新过时的信息

### 5. 报告问题 (Report Issues)

如果你发现问题：

1. **检查是否已存在相关 Issue**
2. **创建新 Issue** 并提供：
   - 清晰的标题
   - 详细的描述
   - 重现步骤（如适用）
   - 预期行为 vs 实际行为

## 代码规范 (Code Standards)

### 创意文件 (Idea Files)

- 使用 Markdown 格式
- 遵循 `TEMPLATE.md` 的结构
- 文件名使用小写字母和连字符
- 包含中英文双语描述

### 项目文档 (Project Documentation)

- 必须包含三个核心文件: README.md, proposal.md, REPOSITORY.md
- 参考 `/Example` 的格式和内容深度
- 保持文档更新与代码同步

### Git Commit 规范 (Git Commit Convention)

使用清晰的 commit 信息：

```
[TYPE] Short description

Detailed description (if needed)
```

**类型 (Types)**:
- `[IDEA]` - 添加或修改创意
- `[PROJECT]` - 添加或更新项目文档
- `[DOCS]` - 文档改进
- `[FIX]` - 错误修复
- `[CHORE]` - 其他维护性工作

**示例 (Examples)**:
```
[IDEA] Add AI-powered code review assistant

[PROJECT] Add blockchain voting system documentation

[DOCS] Improve contributing guidelines

[FIX] Correct typo in README.md
```

## Pull Request 流程 (Pull Request Process)

1. **确保你的更改遵循项目规范**
2. **更新相关文档**（如果需要）
3. **在 PR 中清晰描述你的更改**
4. **等待审核和反馈**
5. **根据反馈进行必要的修改**
6. **获得批准后，PR 将被合并**

### PR 检查清单 (PR Checklist)

在提交 PR 前，请确认：

- [ ] 遵循了文件命名和结构规范
- [ ] 包含了必要的文档
- [ ] Commit 信息清晰明确
- [ ] 没有包含无关的文件更改
- [ ] 中英文双语描述完整（如适用）

## 行为准则 (Code of Conduct)

### 我们的承诺 (Our Pledge)

为了营造一个开放和友好的环境，我们承诺：

- **尊重**所有参与者，无论其经验水平、性别、性取向、残疾、外貌、种族或宗教
- **接纳**不同的观点和经验
- **优雅地接受**建设性批评
- **关注**什么对社区最好
- **表现出同理心**对待其他社区成员

### 不可接受的行为 (Unacceptable Behavior)

- 使用性化的语言或图像
- 侮辱性/贬损性评论，人身或政治攻击
- 公开或私下骚扰
- 未经许可发布他人私人信息
- 其他在专业环境中被认为不当的行为

### 执行 (Enforcement)

不遵守行为准则的参与者可能会被暂时或永久地从项目中移除。

## 投票规则 (Voting Rules)

### 创意投票 (Idea Voting)

- 任何人都可以对 PR 中的创意投票
- 使用 GitHub 的 👍 (thumbs up) 反应
- 每个 GitHub 账户一票
- 不允许刷票或机器人投票

### 评选周期 (Selection Cycle)

- 每月评选一次获得最多投票的创意
- 评选时间：每月最后一天
- 获选创意将被标记并分配给志愿者

### 投票透明度 (Voting Transparency)

- 所有投票都是公开的（GitHub 反应）
- 维护者会定期公布投票统计
- 如发现刷票行为，将取消相关创意的参选资格

## 问题和帮助 (Questions and Help)

如果你有任何问题：

1. **查看 README.md 和文档**
2. **搜索现有的 Issues**
3. **创建新 Issue 提问**
4. **在 Discussions 区域讨论**

## 认可贡献者 (Recognizing Contributors)

我们重视每一位贡献者的努力：

- 所有贡献者都会在项目文档中被提及
- 重要贡献会在 Release Notes 中特别感谢
- 活跃贡献者可能被邀请成为项目维护者
