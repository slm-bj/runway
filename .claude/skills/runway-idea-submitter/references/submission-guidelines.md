# Idea Submission Guidelines

## File Naming Convention

- Use lowercase letters and hyphens only
- Format: `idea-name.md`
- Examples: `ai-code-reviewer.md`, `blockchain-voting-system.md`

## Git Workflow

1. **Create branch**: `git checkout -b idea/your-idea-name`
2. **Create idea file** in `ideas/` directory following template
3. **Commit**: `git commit -m "[IDEA] Add your idea title"`
4. **Push**: `git push origin idea/your-idea-name`
5. **Create PR** with title: `[IDEA] Your Idea Title`

## PR Description Requirements

The Pull Request description must include:
- Brief overview of the idea
- Why this idea is valuable
- Any relevant background information

## Quality Standards

- Idea description must be clear and complete
- Feel free to attach supporting materials (links, images, etc.)
- Focus on AI-aided project ideas (other creative technical projects also welcome)
- One idea per PR
