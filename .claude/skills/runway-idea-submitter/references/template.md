# Idea Template Structure

All idea files must follow this structure from TEMPLATE.md:

```markdown
# Your Idea Title

## Idea Overview

[Briefly describe what your idea is and why it's valuable]

## Detailed Description

[Elaborate on your idea:]
- What problem does it solve?
- How does it solve the problem?
- Why is this approach good?

## Expected Outcomes

[Describe the expected outcomes:]
- User value
- Technical innovations
- Social impact

## Suggested Tech Stack (Optional)

- **Frontend**:
- **Backend**:
- **AI/ML**:
- **Database**:
- **Others**:

## References (Optional)

- [Related articles, projects, or research]

## Contributors

- [@your-github-username] - Idea Author
```

## Content Requirements

### Idea Overview
- Clear, concise description (2-3 sentences)
- Explain what the idea is
- Explain why it's valuable

### Detailed Description
Must address these three questions:
1. **What problem does it solve?** - Identify the pain point or gap
2. **How does it solve the problem?** - Explain the solution approach
3. **Why is this approach good?** - Justify the methodology

### Expected Outcomes
Describe three dimensions:
- **User value**: What benefits do users get?
- **Technical innovations**: What's novel or interesting technically?
- **Social impact**: How does this benefit the community?

### Tech Stack (Optional)
Provide suggestions for:
- Frontend frameworks/libraries
- Backend technologies
- AI/ML components (if applicable)
- Database choices
- Other relevant technologies

### References (Optional)
Link to:
- Research papers
- Similar projects
- Relevant articles
- Documentation

### Contributors
List GitHub username(s) in format: `[@username] - Role`
